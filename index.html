Base structure

/execute is a command that runs another command, but with modifiers. The base structure of new /execute is as follows:

execute run
  • Between execute and run go the modifiers, which will be referred to as subcommands.
  • After the word run goes the command to run, hereafter referred to as the command.
## Old MCBE `/execute` to new `/execute` ## Old MCBE `/execute` is simply the same as new `/execute`, but with *fixed* subcommands. That is, you do not get to choose which subcommands to apply, you must only use the subcommands in a certain order. Every old `/execute` command can be turned into a new `/execute` command like so: execute @e[tag=test] ~12 ~23 ~34 summon lightning_bolt execute as @e[tag=test] at @s positioned ~12 ~23 ~34 run summon lightning_bolt execute @e[tag=test] ~12 ~23 ~34 detect ~ ~-1 ~ stone 0 summon lightning_bolt execute as @e[tag=test] at @s positioned ~12 ~23 ~34 if block ~ ~-1 ~ stone 0 run summon lightning_bolt Or, in general form, execute run execute as at @s positioned run execute detect execute as at @s positioned if block run With new `/execute`, you are no longer restricted to these two variants presented above. You can choose any number of subcommands, in any order, to modify the way the command is run.
